## 2024-05-18 - Improve Interactive List Accessibility
**Learning:** Interactive lists in this application often use `motion.div` with an `onClick` handler. This approach creates an inaccessible list item for keyboard and screen reader users, because `div` elements are not natively focusable or identifiable as interactive controls.
**Action:** When building interactive list items, use `<motion.button>` (or a native `<button>` element) instead of a `div`. This provides native keyboard accessibility (tab focus, enter/space activation). Ensure to apply appropriate ARIA attributes like `aria-pressed` or `aria-selected` to convey state, and styling adjustments (e.g., `width: '100%'`, `textAlign: 'left'`, `border: 'none'`, `background: 'none'`) to make the button look like a list item.
