## 2024-05-14 - Unauthenticated Paywall Bypass API Endpoint
**Vulnerability:** A development-only `/api/unlock_dev` endpoint remained in the FastAPI backend (`backend/main.py`). This endpoint accepted any `session_id` and added it to the `unlocked_sessions` set without authentication or authorization, allowing anyone to bypass the Stripe premium paywall completely. The Next.js frontend (`frontend/src/app/page.tsx`) actively used this endpoint via an "UNLOCK PREMIUM" button instead of a proper Stripe integration.
**Learning:** Development convenience routes or mock handlers that bypass core business logic (like payments or auth) are frequently left in production code if they are not conditionally disabled based on the environment (e.g., `if os.getenv("ENV") == "development":`). Furthermore, if the frontend is built to depend on these mock routes, removing the backend route requires a coordinated change to prevent breaking the UI.
**Prevention:** Avoid committing "dev-only" endpoints that bypass security or payment checks without strict, environment-level safeguards. When building features that require payments (like Stripe checkout), implement at least a placeholder integration or a mock that respects the intended authorization flow from the beginning. Never wire a UI button to a "free unlock" API in production-bound code.
